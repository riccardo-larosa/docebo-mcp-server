Connect in seconds

Works with any MCP client.

OAuth 2.0 authentication built in. No SDK, no custom code.

Claude Desktop

Add a remote MCP server in settings. Claude handles OAuth authorization automatically.

https://your-instance.mcp.yourdomain.com/mcp

Claude Code

One command to connect. Authenticates via your browser.

claude mcp add docebo --transport http https://your-instance.mcp.yourdomain.com/mcp

Other MCP clients

Cursor, Windsurf, and any client that supports remote MCP servers with OAuth 2.0.

https://your-instance.mcp.yourdomain.com/mcp
How it works

Three layers. Zero complexity.

The MCP server translates natural-language tool calls into Docebo API requests — handling authentication, validation, pagination, and error formatting.

Secure by design

Enterprise-grade security, built in.

No API keys stored. No elevated permissions. Just OAuth 2.0 and your existing access controls.

OAuth 2.0

Industry-standard authentication. Users authorize with their own Docebo credentials. No API keys stored on the server.

Per-user permissions

The AI can only access what the authenticated user can access in Docebo. Your existing role-based access controls apply.
